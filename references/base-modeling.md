# 飞书 Base 建模指南

## 推荐表结构

| 表名 | 粒度 | 说明 |
| --- | --- | --- |
| `00_输入数据目录` | 文件/工作表 | 来源文件、sheet、行列数、时间范围、导入时间 |
| `01_字段字典与质量检查` | 字段 | 字段类型、缺失率、唯一值数、异常说明 |
| `02_分析问题与指标口径` | 指标/问题 | 业务问题、指标定义、公式、过滤条件 |
| `03_清洗转换步骤` | 处理步骤 | 步骤编号、输入、动作、输出、影响 |
| `04_核心明细或宽表` | 明细行 | 清洗合并后的可复核底表 |
| `05_聚合结果表` | 指标维度组合 | 按维度聚合的核心结果 |
| `06_洞察与行动建议` | 洞察 | 结论、证据、建议、优先级、验证指标 |

可根据实际主题增加表，例如 `07_异常明细`、`08_客户分层`、`09_渠道ROI`。

## 字段类型建议

| 数据语义 | Base 字段类型 |
| --- | --- |
| 名称、编号、说明 | 文本 |
| 金额、数量、比例、得分 | 数字 |
| 日期、月份、时间戳 | 日期时间 |
| 状态、等级、优先级 | 单选 |
| 标签、归因原因 | 多选 |
| 是否异常、是否有效 | 复选框 |

写记录前先读 `lark-base-shortcut-record-value.md`，保证字段值结构正确。

## 原子化记录要求

- 每个清洗/转换步骤都有唯一 `步骤编号`，例如 `S01`、`S02`。
- 每个结果表都能追溯到 `输入表` 和 `计算口径`。
- 不把多个业务问题混在一张结果表里；一张表只服务一个清晰分析目的。
- 对大明细表保留必要字段，避免把无关字段全部塞入 Base。

## 创建顺序

1. 创建 Base：读 `lark-base-base-create.md`。
2. 创建子表：读 `lark-base-table-create.md`。
3. 创建字段：读 `lark-base-field-create.md` 和 `lark-base-shortcut-field-properties.md`。
4. 写入记录：读 `lark-base-record-batch-create.md` 和 `lark-base-shortcut-record-value.md`。
5. 创建视图：按需读 `lark-base-view-create.md`、筛选/排序/分组/可见字段文档。
6. 创建看板：读 dashboard 相关文档。

## 批量写入约束

- `+record-batch-create` 单次最多 200 行。
- 字段顺序和每行数组顺序必须一致。
- 空值用 `null`。
- 表结构和记录写入不要并行操作同一张表。
- 对原始大表，如果行数过大，优先只写抽样、汇总、异常明细和可复核关键底表；完整原始文件可在文档中说明来源。
