# 认证、权限与安全规则

本技能通过 `lark-cli` 创建飞书多维表格和飞书云文档。执行前先确认本机已有 `lark-cli`，并尽量使用最小必要权限。

## 身份选择

`lark-cli` 支持两类身份：

| 身份 | 参数 | 适用场景 |
| --- | --- | --- |
| 用户身份 | `--as user` | 创建在用户云空间、访问用户已有资源、需要用户视角权限 |
| 应用身份 | `--as bot` | 应用自有资源、自动化创建资源 |

若用户希望产物出现在自己的云空间，优先使用 `--as user`。如果以 bot 创建，注意创建结果里的 `permission_grant`，确认是否已把可管理权限授给当前用户。

## 首次配置

如果 `lark-cli` 未配置，运行：

```bash
lark-cli config init --new
```

该命令会输出授权链接，需要用户打开并完成配置。

## 用户授权

权限不足时，错误里通常包含缺失 scope 和修复建议。用户身份授权使用：

```bash
lark-cli auth login --domain <domain>
lark-cli auth login --scope "<missing_scope>"
```

必须指定 `--domain` 或 `--scope`，不要无范围授权。

## 写入确认

本技能会创建 Base、创建表、写记录、创建仪表盘、创建 Doc。若用户本轮明确要求“把分析过程和结论产物沉淀到飞书中”，视为已授权执行这些创建动作；若用户只是询问方案或评估可能性，不要实际写入。

## 安全规则

- 禁止输出 appSecret、accessToken 等密钥。
- 遇到权限错误时，向用户说明缺少的 scope 和后台链接，不要猜测或绕过权限。
- 删除、覆盖、迁移 owner 等高风险操作必须单独确认。
- 对同一 Base 的结构修改和仪表盘组件创建应串行执行，避免状态竞争。
