# 业务数据分析 Playbook

## 分析设计原则

1. 先回答业务问题，再选择算法或图表。
2. 所有结论都要能追溯到输入表、过滤条件和计算口径。
3. 把探索、清洗、计算、结论分开沉淀，避免把过程隐藏在代码里。
4. 对异常值、缺失值、重复记录、口径边界单独说明，不让它们悄悄影响结论。

## 常见分析框架

| 场景 | 方法 | 常见输出表 | 常见图表 |
| --- | --- | --- | --- |
| 经营概览 | KPI 拆解、同比/环比、贡献度 | 指标总览、趋势表、贡献排名 | 指标卡、折线图、柱状图 |
| 销售/收入 | 漏斗、渠道 ROI、客单价、复购 | 渠道表现、区域排行、商品排行 | 漏斗图、条形图、组合图 |
| 用户/客户 | RFM、分群、留存、转化 | 客户分层、 cohort 留存、流失预警 | 热力表、折线图、散点图 |
| 产品/运营 | 转化路径、功能使用、活跃度 | 路径转化、功能渗透、异常清单 | 漏斗图、趋势图、帕累托图 |
| 财务/成本 | 成本结构、毛利、预算偏差 | 成本分类、毛利分析、偏差表 | 瀑布思路表、饼图、柱状图 |
| 供应链/履约 | 时效、库存周转、异常履约 | 履约时长、库存周转、异常明细 | 箱线思路表、折线图、排名图 |

## 数据体检清单

- 行数、列数、工作表名、文件来源。
- 字段类型：文本、数字、日期、分类、布尔、金额、比例。
- 缺失值：缺失率、关键字段缺失明细。
- 重复值：主键候选、重复主键、重复明细。
- 异常值：负数、极端值、日期越界、状态枚举异常。
- 时间范围：最早日期、最晚日期、粒度（日/周/月）。
- 可关联性：跨表 join 键、匹配率、未匹配样本。

## 指标口径写法

每个指标都在 `02_分析问题与指标口径` 中记录：

| 字段 | 说明 |
| --- | --- |
| 指标名称 | 如 GMV、净收入、转化率、复购率 |
| 业务问题 | 指标回答什么问题 |
| 分子 | 转化率、占比类指标的分子 |
| 分母 | 转化率、占比类指标的分母 |
| 过滤条件 | 时间、状态、渠道、有效订单等 |
| 计算公式 | 可读公式或 SQL/pandas 表达式 |
| 口径边界 | 是否含退款、取消、空值、异常值 |
| 对应结果表 | Base 中的表名 |

## 洞察写法

每条洞察建议按这个结构沉淀到 `06_洞察与行动建议`：

- `结论`：一句话说明发现。
- `证据`：引用结果表、指标、Top N、趋势变化。
- `解释`：为什么可能发生，说明是否为推断。
- `影响`：涉及金额、用户数、转化率、成本或风险。
- `建议动作`：下一步做什么。
- `优先级`：P0/P1/P2。
- `验证指标`：后续如何判断动作有效。

## 报告叙事顺序

1. 先给执行摘要，让用户知道最重要的结论。
2. 再说明数据来源和可信度。
3. 按业务问题逐节展开分析。
4. 每节先讲图表/结果，再讲解释和建议。
5. 结尾给行动清单和后续追踪指标。
